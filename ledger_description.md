# Ledger Description

## Purpose

`ledger.csv` is the running log of all work performed during the project. Every working session, meeting, and file change is recorded here with enough detail to reconstruct the project timeline.

## Column Definitions

| Column | Description |
|--------|-------------|
| `Date` | Date the work was performed (YYYY-MM-DD) |
| `Member` | Team member(s) involved (name or "All") |
| `Meeting/Task` | Short title for the task or meeting |
| `Description` | 1–3 sentence summary of what was done and why |
| `Files Changed` | Files created or modified during this task |
| `Output` | Artifact produced (plot, CSV, model, document) |
| `Status` | Completed / In Progress / Blocked |

## Update Rules

- At least one entry is added after every working session
- The ledger must have a minimum of 10 entries by project end
- Entries where AI coding assistance was used are noted in the Description field
- All team members are responsible for keeping the ledger current
